Como publicar sua página no GitHub, conectar um domínio, integrar o Firebase e editar tudo remotamente usando o Claude AI.

01
Preparar o arquivo HTML
Sua página pronta para publicar
›

Você precisa ter sua página em um único arquivo index.html. Tudo — CSS, JS e HTML — dentro desse arquivo.

💡
Se sua página usa vários arquivos (style.css, script.js), use o Claude para juntar tudo em um único index.html.
Prompt para o Claude
"Tenho uma página HTML com arquivos separados de CSS e JS. Junte tudo em um único arquivo index.html, com o CSS no <style> e o JS no <script>, sem perder nenhuma funcionalidade."
✅
Resultado: um único arquivo index.html pronto para subir no GitHub.
02
Criar conta e repositório no GitHub
github.com → New repository
›

Acesse github.com e crie sua conta se ainda não tiver. Depois:

  • Clique em "New repository" (botão verde)
  • Nome do repositório: pode ser o nome do seu projeto (ex: eletricista-power)
  • Marque como Public
  • Marque "Add a README file"
  • Clique "Create repository"
💡
O nome do repositório vai aparecer na URL. Escolha algo sem espaços e sem caracteres especiais.
03
Subir o index.html no GitHub
Usando o github.dev — sem instalar nada
›

No seu repositório, troque .com por .dev na URL do navegador:

github.com/seuusuario/seurepositorio
↓
github.dev/seuusuario/seurepositorio

Vai abrir um VS Code no navegador. Então:

  • Clique no ícone de arquivo (canto superior esquerdo)
  • Arraste seu index.html para dentro do explorador de arquivos
  • Ou clique com botão direito → "Upload Files"
  • Clique no ícone do Git (ramificação na barra lateral)
  • Escreve qualquer mensagem (ex: "primeiro upload")
  • Clique "Commit & Push"
✅
Seu arquivo já está no GitHub!
04
Ativar o GitHub Pages
Publicar a página gratuitamente
›

No seu repositório no GitHub:

  • Clique em Settings (aba no topo)
  • No menu lateral esquerdo: Pages
  • Em "Source" selecione: Deploy from a branch
  • Branch: main → pasta: / (root)
  • Clique Save

Em 1-2 minutos sua página vai estar disponível em:

https://seuusuario.github.io/seurepositorio
💡
A cada vez que você fizer Commit & Push, o GitHub Pages republica automaticamente em ~2 minutos.
05
Conectar domínio personalizado
Ex: eletricistapro.com.br
›

Se você tem um domínio (ex: eletricistapro.com.br), pode apontar para o GitHub Pages:

No painel do seu provedor de domínio (Registro.br, GoDaddy, etc), adicione esses registros DNS:

Registros DNS — tipo A
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
Registro CNAME (para www)
www → seuusuario.github.io

No GitHub (Settings → Pages → Custom domain):

  • Digite seu domínio: eletricistapro.com.br
  • Marque "Enforce HTTPS"
⏳
DNS pode demorar até 24h para propagar. Enquanto isso sua página continua no endereço github.io.
06
Adicionar modo de edição com Claude AI
Peça ao Claude para implementar
›

Cole seu index.html no Claude e use esse prompt:

Prompt — Modo de edição completo
"Essa página está no modo de edição. Tire todas as ferramentas de edição visíveis e implemente um sistema completo onde: textos com data-editable sejam editáveis, emojis com data-icon sejam trocáveis, preços com data-price sejam editáveis, botões de WhatsApp sejam configuráveis. Tudo protegido por senha. O modo de edição só abre após digitar a senha correta."
💡
Você pode fazer isso com qualquer página HTML — não precisa ter sido feita pelo Claude.
07
Criar projeto no Firebase
Banco de dados gratuito na nuvem
›

Acesse console.firebase.google.com:

  • Clique "Adicionar projeto" → nome do projeto → criar
  • No menu: Criação → Realtime Database
  • Clique "Criar banco de dados"
  • Escolha "Iniciar no modo de teste" → Criar
  • Copie a URL que aparece: https://seuprojeto-default-rtdb.firebaseio.com

Depois vá em ⚙️ Configurações do projeto → Seus apps → Web </>:

  • Registre um app web → copie o bloco firebaseConfig
⚠️
O modo de teste expira em 30 dias. Para produção, configure as regras de segurança no Firebase.
08
Integrar Firebase na página
Edições salvas na nuvem
›

Com o firebaseConfig em mãos, peça ao Claude:

Prompt — Integrar Firebase
"Implemente o Firebase Realtime Database nessa página. O Firebase deve carregar APENAS quando o usuário digitar a senha do modo de edição — os visitantes normais não devem carregar o SDK. Use esse config: [cole seu firebaseConfig aqui]. As edições devem ser salvas na nuvem e carregadas automaticamente quando entrar no modo edição."
✅
Resultado: visitantes não pagam o peso do Firebase. Só você carrega quando vai editar.
09
Autorizar domínio no Firebase
Evita erro de domínio não autorizado
›

O Firebase bloqueia requisições de domínios desconhecidos. Você precisa autorizar o seu:

  • Firebase Console → Authentication → Começar
  • Aba Settings → Domínios autorizados
  • Clique "Adicionar domínio"
  • Adicione: seudominio.com.br
  • Adicione: www.seudominio.com.br
  • Adicione também: seuusuario.github.io
💡
Adicione sem http:// e sem barra no final. Só o domínio limpo.
10
Subir e editar remotamente
Fluxo final de uso
›

Com tudo configurado, seu fluxo de edição fica assim:

📱 Abre sua página no celular ou PC
✏️ Clica no botão Modo Edição
🔑 Digita a senha
☁️ Firebase carrega seus dados
💾 Edita e salva na nuvem
✅ Qualquer dispositivo vê atualizado
🎉
Pronto! Você pode editar sua página de qualquer lugar, sem baixar arquivos, sem mexer no GitHub, sem complicação.

O fluxo em 5 passos
1

Prepare seu index.html

CSS, JS e HTML tudo num único arquivo. Se tiver separado, peça ao Claude para juntar.

2

Suba no GitHub + ative o Pages

Crie um repositório público. Acesse github.dev/usuario/repo, faça upload do arquivo e commit. Ative o GitHub Pages em Settings → Pages.

github.dev/seuusuario/seurepositorio
3

Conecte seu domínio (opcional)

Aponte os DNS para o GitHub e configure o domínio em Settings → Pages → Custom domain.

4

Integre o Firebase com Claude

Crie um projeto no Firebase, ative o Realtime Database, copie o firebaseConfig e peça ao Claude para integrar na página com carregamento lazy e senha de proteção.

console.firebase.google.com
5

Edite de qualquer lugar

Acesse sua página, clique ✏️, digite a senha → Firebase carrega → edite → salve na nuvem. Qualquer dispositivo vê atualizado.

Como editar remotamente
→

Abra sua página publicada

No celular, tablet ou computador — qualquer navegador.

→

Clique no botão ✏️ Modo Edição

Fica fixo no canto da tela. Só aparece para você.

→

Digite a senha

O Firebase carrega automaticamente seus dados da nuvem.

→

Edite o que quiser

Clique em textos para editar. Clique em emojis para trocar. Clique em preços para atualizar. Clique em botões para configurar WhatsApp.

→

Clique 💾 Salvar Edições

Tudo vai para a nuvem Firebase. Abra em outro dispositivo — já está atualizado.

Dicas importantes
⚠️

Firebase modo de teste expira em 30 dias

Configure as regras de segurança antes de 30 dias ou o banco vai bloquear acesso.

⚠️

Autorize seu domínio no Firebase

Firebase → Authentication → Settings → Domínios autorizados. Adicione sem http:// e sem barra.

💡

DNS demora até 24h para propagar

Após apontar o domínio para o GitHub, aguarde. Sua página já funciona no endereço github.io.

💡

Edição no celular funciona 100%

O modo edição foi feito para mobile. Você pode editar sua página em qualquer lugar.
